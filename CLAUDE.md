# Dual Finder 纪 协作记忆

## 编码与发布偏好

- 每次编码和测试完成后，如果这是 mac 应用，需要运行或提示运行 `./update_app.sh`，确保重新编译、ad-hoc 签名、复制到 `/Applications` 并从 `/Applications` 启动。
- 每次具备发布条件后，使用 `gh` 更新 GitHub 仓库、推送代码并更新 GitHub Releases。
- mac 应用必须保留持久日志，默认每日一个文件、最多 7 天，不随重启清空。
- mac 应用关闭窗口后应退出进程；启动应单实例并最大化。
- 发布前需要重新审视修改代码、测试覆盖、边界场景、可维护性、分层和 DRY。
